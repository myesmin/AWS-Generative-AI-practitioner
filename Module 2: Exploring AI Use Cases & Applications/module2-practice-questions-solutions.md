### Question 1

**A pharmaceutical company wants to use AI to generate new potential molecular structures for drug candidates and reduce research costs. Which industry application of AI does this represent?**

- A) Financial Services — portfolio management
- B) Life Sciences — drug discovery ✅
- C) Manufacturing — predictive maintenance
- D) Retail — pricing optimization

> **Explanation:** Generating molecular structures for drug candidates is a Life Sciences use case — specifically drug discovery. AI accelerates this process by generating and screening potential compounds far faster than traditional lab methods, reducing both cost and time to discovery.

---

### Question 2

- A) Process optimization
- B) Product design
- C) Predictive maintenance ✅

## Section 2: When AI and ML Are Appropriate

---

### Question 13

**A developer needs to calculate sales tax on a transaction. The rule is straightforward: multiply the total by a fixed percentage. Should they use ML for this?**

- A) Yes — ML is always more accurate than rule-based computation
- B) Yes — ML can handle tax calculations at scale
- C) No — this can be solved with a simple computation; ML is unnecessary ✅
- D) No — taxes require deep learning, not standard ML

> **Explanation:** The course is clear: you do not need ML if you can determine a target value using simple rules, computations, or predetermined steps. Multiplying by a fixed percentage is a simple formula — no data-driven learning is needed.

---

### Question 14

**A company receives 50 million customer emails per day and wants to automatically classify them as urgent or non-urgent to prioritize support responses. Which statement best justifies using ML for this task?**

- A) ML is unnecessary because email classification is a simple task
- B) ML is appropriate because manually classifying millions of emails is infeasible, and patterns are too complex for static rules ✅
- C) ML should not be used because email data is unstructured
- D) ML is only appropriate if all emails have been pre-labeled

> **Explanation:** Two conditions from the course are met: (1) scale — 50 million emails daily makes human review impossible; and (2) complexity — the rules for what makes an email "urgent" involve many overlapping factors that are difficult to code manually.

---

### Question 15

**A small business owner receives around 50 support tickets per week. Their team manually reads each one and routes it using a fixed decision tree with 5 clear rules. Should they replace this with an ML model?**

- A) Yes — ML always performs better than rule-based systems
- B) Yes — ML should be used for any classification task
- C) No — the task is small scale and well-defined rules already work effectively ✅
- D) No — ML cannot be used for support ticket routing

> **Explanation:** The course states that when rules can be determined by simple, clear, predefined steps, ML is unnecessary. 50 tickets per week with 5 clear rules is manageable with rule-based logic. ML would add unnecessary complexity and cost.

---

### Question 16

**Which of the following situations best indicates that ML is the RIGHT choice?**

- A) A target value can be calculated with a predetermined formula
- B) A problem involves a small, stable dataset with clearly defined rules
- C) Rules are too complex, overlapping, or require too many factors to manually code ✅
- D) A system needs to follow compliance regulations with no ambiguity

> **Explanation:** The course states ML is a good choice when coding rules is a challenge — specifically when rules rely on too many factors, have overlaps, or need to be finely tuned. Options A and B describe rule-based scenarios; option D describes a compliance requirement better served by rules.

---

### Question 17

**A bank wants to detect fraud across 100 million daily transactions. A human analyst can review about 500 transactions per day. Which characteristic of this problem makes ML the appropriate solution?**

- A) The transactions are structured data
- B) The scale of the problem makes human review infeasible ✅
- C) The bank has labeled fraud data available
- D) Fraud detection requires deep learning specifically

> **Explanation:** The course cites scale as a key reason to use ML. Humans reviewing 100 million transactions daily at 500 per day would take 548 years per day of data. ML scales to millions of transactions automatically — this is the primary justification for ML here.

- A) Classifying millions of images as containing cats or dogs

## Section 3: ML Techniques and Use Cases

---

### Question 19

**A data scientist is building a model to predict the selling price of a house based on square footage, number of bedrooms, and location. Which ML technique should they use?**

- A) Clustering
- B) Dimensionality reduction
- C) Regression ✅
- D) Classification

> **Explanation:** Predicting a continuous numerical value (house price) is a regression problem. Regression models the relationship between input features and a continuous output. Classification would be used if the output were a category (e.g., "above $500k" vs. "below $500k").

---

### Question 20

**A bank's ML model analyzes each incoming transaction and outputs either "fraudulent" or "legitimate." Which supervised learning subcategory does this represent?**

- A) Regression
- B) Clustering
- C) Dimensionality reduction
- D) Classification ✅

> **Explanation:** Outputting a discrete category label (fraudulent or legitimate) is classification — a supervised learning subcategory. The model was trained on labeled transactions (fraud/not fraud) and now predicts the label for new transactions.

---

### Question 21

**A marketing team has millions of customer purchase records with no predefined segments. They want the AI system to automatically discover groups of customers with similar buying behaviors. Which ML technique is most appropriate?**

- A) Regression
- B) Classification
- C) Clustering ✅
- D) Reinforcement learning

> **Explanation:** Automatically discovering natural groupings in unlabeled data is clustering — an unsupervised learning technique. No predefined categories exist; the algorithm finds similarity patterns and groups customers accordingly. This is the customer segmentation use case from the course.

---

### Question 22

- A) Classification

- C) Clustering
- D) Dimensionality reduction ✅

> **Explanation:** Dimensionality reduction reduces the number of features in a dataset while preserving the most important information. Big data visualization is explicitly listed as a use case in the course. Techniques like PCA (Principal Component Analysis) are common examples.

- A) Predicting the next quarter's revenue
- B) Grouping customers into market segments

### Question 25

**Which of the following is a use case for regression in supervised learning?**

- A) Detecting whether an email is spam or not spam
- B) Grouping news articles by topic
- C) Forecasting the expected population growth over the next 10 years ✅
- D) Identifying anomalous customer accounts

> **Explanation:** Population growth forecasting predicts a continuous numerical value over time — this is regression. Option A is classification (two categories). Option B is unsupervised clustering. Option D is anomaly detection, typically unsupervised.

### Question 26

**What is the key characteristic that distinguishes supervised learning from unsupervised learning?**

- A) Supervised learning uses neural networks; unsupervised does not
- B) Supervised learning requires labeled training data; unsupervised learning does not ✅
- C) Supervised learning is always more accurate than unsupervised learning
- D) Supervised learning can only be used for image data

> **Explanation:** The fundamental distinction is labels. Supervised learning requires labeled data — each example comes with a known correct output. Unsupervised learning has no labels; the algorithm discovers patterns in the data on its own.

### Question 28

**In reinforcement learning, what role does the "reward function" play?**

- A) It labels training data for the agent to learn from
- B) It provides incentives or penalties to guide the agent toward the desired behavior ✅

### Question 34

**A company deployed a generative AI content system. A data scientist ran the same prompt five times and received five noticeably different outputs each time. Which challenge of generative AI is this an example of?**

- A) Toxicity
- B) Hallucination
- C) Social risks
- D) Nondeterminism ✅

> **Explanation:** Nondeterminism is the challenge where the model generates different outputs for the same input. The course states this "can cause problems in applications where reliability is key." Mitigation: run the model multiple times and compare outputs to identify inconsistency.

---

### Question 35

**A generative AI model used in a medical information app confidently states that a particular medication has no side effects, when in fact it has several documented ones. Which challenge does this represent?**

- A) Nondeterminism
- B) Toxicity
- C) Hallucination ✅
- D) Interpretability

> **Explanation:** A hallucination is when the model generates inaccurate responses that are not consistent with reality or training data. The model confidently producing false medical information is a classic hallucination — particularly dangerous in healthcare contexts.

## Key Exam Traps to Watch For

```
❗ Capabilities vs. Metrics:
   Personalization, Scalability, Simplicity = CAPABILITIES
   Conversion Rate, ARPU, Cross-Domain = METRICS

❗ Regression vs. Classification:
   Predict a NUMBER  → Regression
   Predict a CATEGORY → Classification

❗ Clustering vs. Classification:
   Labels provided → Classification (supervised)
   No labels, find groups → Clustering (unsupervised)

❗ IDP vs. NLP vs. Computer Vision:
   Extract data from scanned documents/forms → IDP
   Understand or classify text content → NLP
   Analyze images or video → Computer Vision

❗ Hallucination vs. Nondeterminism:
   False but confident outputs → Hallucination
   Inconsistent outputs for same input → Nondeterminism

❗ When NOT to use ML:
   Simple formula → Direct computation
   Clear rules, small scale → Rule-based logic
```
